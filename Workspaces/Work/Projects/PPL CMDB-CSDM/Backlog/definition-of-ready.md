# Definition of Ready — CMDB-CSDM

A story is ready to be pulled into Sprint Planning when all of the following gates have been passed in order. See [[requirements-process]] for the full intake flow and [[definition-of-done]] for acceptance criteria.

## Gate 1 — Requirements Confirmed

- [ ] PO (Joe Dames) has met with stakeholders (PPL / Accenture) to gather and confirm requirements
- [ ] OR request has been submitted via Application Services Employee Center and reviewed with Uloma (PPL BA)

## Gate 2 — Story Created in ADO Correctly

- [ ] Story created under the correct Epic > Feature hierarchy in ADO
- [ ] Created using the "+" icon from the Epic/Feature link (ensures correct linkage — never create standalone)
- [ ] Epic and Feature are owned by Joe Dames (PO) — PO owns all Epic and Feature levels
- [ ] Story ID shared with Anu for Program Backlog Review tracking
- [ ] Priority assigned: **P1** (Must Have) / **P2** (Should Have) / **P3** (Could Have) / **P4** (Nice to Have)
- [ ] If multiple P1 stories exist, PO has provided numeric sub-ranking (1, 2, 3...) for Sprint Planning execution order

## Gate 3 — Team Backlog Refinement Passed

- [ ] Story reviewed at **Service Mapping / CSDM / CMDB Backlog Refinement** with Dev team
- [ ] Story sized (story points agreed by team)
- [ ] Acceptance criteria clear to BPC, Tester, and Developers
- [ ] Technical solution discussed and understood

## Gate 4 — Program Backlog Review Passed

- [ ] Story reviewed at **SERVNW Program Backlog Refinement & Prioritization** (Wednesday)
- [ ] Anu confirms story is ready for Sprint Planning

## Gate 5 — Sprint Planning Ready

- [ ] No unresolved blockers or dependencies
- [ ] Story fits within one iteration
- [ ] PO present to answer questions at Sprint Planning
