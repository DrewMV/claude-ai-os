# NERCipedia — CIP-010-4

**URL**: https://nercipedia.com/active-standards/cip-010-4-cyber-security-configuration-change-management-and-vulnerability-assessments/

## What It Covers

Authoritative reference for the current active version of NERC CIP-010 (version 4). Documents all four requirements (R1–R4) with their sub-parts, applicability tables, and violation risk factors.

## Key Claims

- **R1.1**: Baseline must include OS/firmware, commercially available software, custom software, logical ports, and security patches
- **R1.2**: Entities must "authorize and document changes that deviate from the existing baseline configuration"
- **R1.4**: Only controls "that could be affected" by the specific change need to be verified — not all CIP-005/CIP-007 controls
- **R1.5**: Emergency change procedures must exist for situations requiring immediate response
- **R2**: Unauthorized modifications must be detected — the standard implies a 35-day cycle (referenced in related guidance)
- **R3**: Vulnerability assessments required; scope varies by impact level
- **R4**: Transient cyber assets require authorization, software validation, and malicious code mitigation
- **Evidence retention**: 3 calendar years minimum for all requirements
- **Violation Risk Factor**: Medium for all four requirements
