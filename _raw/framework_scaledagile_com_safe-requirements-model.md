# SAFe Requirements Model

> Essentially, all models are wrong, but some are useful.
>
> —George E. P. Box

To support bringing the benefits of Lean and Agile development to larger enterprises—or smaller businesses building more complex systems—SAFe provides a scalable requirements model that demonstrates a way to express system behaviors: Epics, Capabilities, Features, Stories, Nonfunctional Requirements (NFRs), and more. Each of these work items is expressed differently.
